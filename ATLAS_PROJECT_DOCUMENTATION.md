# The Atlas 项目技术文档

## 1. 项目概述

**The Atlas** 是一个强大的程序化生成引擎，专注于创建庞大、复杂的星系和宇宙。它受到 No Man's Sky 的启发，通过先进的算法和真实的天体物理现象模拟，提供了身临其境的宇宙探索体验。

### 核心特性

- **🌌 沉浸式 3D 探索**：使用 Three.js 实现行星、恒星系统和星系的全 3D 可视化
- **⛏️ 游戏挖矿系统**：手动和自动资源提取，带有冷却时间和每日限制
- **💎 三种核心资源**：反物质、115 号元素和氘，用于为飞船提供动力
- **🚀 飞船升级系统**：带有效率、范围和乘数升级的等级系统
- **🎯 每日挑战**：难度随时间增加的可扩展目标
- **💰 秘密宝箱**：超高稀有度的宝藏，生成概率为 0.000001
- **🧬 10 种生命形式**：从智能生命到神圣实体
- **🌍 26+ 行星类型**：每种都有独特的特征和特定资源
- **🔮 行星异常现象**：稀有而神秘的现象等待发现
- **📱 截图功能**：生成 4K 截图，包括即时访问和图像共享的二维码
- **🌀 星门链接**：用于即时旅行的编码 URL，可与朋友分享任何位置

## 3. P2P 网络系统

### 核心组件

1. **AtlasP2PRoutes**：主要的 P2P 路由管理类，处理节点间通信
2. **AtlasP2PDiscovery**：基于 BitTorrent DHT 的节点发现系统
3. **KnownPeerManager**：管理已知的 P2P 节点
4. **FailedPeerManager**：管理失败或恶意的节点
5. **AtlasCryptographicManager**：处理加密和安全验证

### 连接流程

1. **初始化 P2P 系统**：创建必要的目录结构，初始化加密管理器，启动 DHT 发现服务
2. **节点发现**：使用 BitTorrent DHT 协议发现网络中的其他节点
3. **节点握手**：与发现的节点进行握手，交换身份和宇宙数据
4. **数据同步**：交换宇宙配置信息，广播消息到网络中的其他节点

### 安全措施

1. **代码完整性验证**：通过比较代码哈希确保所有节点运行相同版本的代码
2. **时间戳验证**：防止重放攻击
3. **节点身份验证**：使用加密身份标识
4. **恶意节点检测**：检测和封禁可疑节点
5. **失败节点管理**：记录和管理连接失败的节点

## 4. 部署指南

### Docker 部署（推荐）

1. **创建 docker-compose.yml 文件**
   ```yaml
   version: "3.9"
   services:
     the-atlas:
       image: bansheetech/atlas:latest
       container_name: the-atlas
       ports:
         - "42424:42424"
       environment:
         - EXTERNAL_P2P_OPEN_PORT=42424
         - ENABLE_P2P=true
       volumes:
         - ./atlas_docker_volume:/app
       restart: always
   ```

2. **启动容器**
   ```bash
   docker-compose up -d
   ```

3. **验证部署**
   - 访问 `http://localhost:42424` 完成初始化
   - 检查 P2P 状态：`http://localhost:42424/api/p2pv2/status`

### 直接部署

1. **克隆仓库**
   ```bash
   git clone https://github.com/SurceBeats/Atlas.git
   cd Atlas
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **创建必要的目录**
   ```bash
   mkdir -p internal_data/p2p internal_data/logs
   ```

4. **启动服务器**
   ```bash
   python __main__.py
   ```

### 网络要求

1. **端口开放**：必须开放 TCP/UDP 端口 42424
2. **NAT 穿透**：如果服务器位于 NAT 后面，需要配置端口转发
3. **网络连接**：服务器需要稳定的互联网连接

## 5. 游戏玩法重构方案

### 现状分析

当前 The Atlas 游戏拥有以下核心玩法系统：

- **资源系统**：三种核心资源（反物质、115号元素、氘）
- **飞船系统**：等级、效率、范围、存储、乘数升级
- **挖矿系统**：手动和自动资源提取
- **旅行系统**：基于距离的资源消耗
- **被动资源生成**：基于已发现的行星和系统
- **资源交换**：不同资源之间的兑换

### 重构目标

1. **增加游戏深度**：引入更多玩法维度和战略选择
2. **提升玩家参与度**：增加目标和成就感
3. **扩展游戏内容**：增加可探索的内容和活动
4. **增强社交互动**：利用现有的 P2P 网络
5. **保持核心体验**：保留宇宙探索的核心乐趣

### 核心玩法循环重构

**当前循环**：探索 → 挖矿 → 升级飞船 → 探索更远

**重构后循环**：
1. **探索**：发现新的宇宙、星系、系统和行星
2. **采集**：获取资源和特殊物品
3. **加工**：将原始资源转化为高级产品
4. **交易**：与其他玩家或 NPC 交易
5. **建造**：在行星上建立基地和设施
6. **研究**：解锁新科技和能力
7. **征服**：完成挑战和任务
8. **协作**：与其他玩家合作完成大型项目

### 资源系统扩展

#### 资源分层
- **基础资源**：反物质、115号元素、氘（保持不变）
- **中级资源**：从基础资源合成的高级材料
- **稀有资源**：特定行星类型产出的独特资源
- ** artifact资源**：从古代遗迹或异常现象中发现的神秘物品

#### 资源用途扩展
- **飞船燃料**：用于旅行
- **建筑材料**：用于基地建设
- **科研材料**：用于解锁科技
- **交易商品**：用于经济系统
- **任务物品**：用于完成特定任务

### 飞船系统改进

#### 飞船类型
- **探索舰**：高速度、长距离，适合探索新区域
- **采矿舰**：高存储、高效率，适合资源采集
- **战斗舰**：高防御、高武器，适合应对危险
- **贸易舰**：高容量、低消耗，适合长途贸易
- **科研舰**：高扫描能力，适合发现稀有资源和异常

#### 飞船定制
- **模块系统**：可安装不同功能的模块
- **外观定制**：可自定义飞船外观
- **技能树**：每种飞船类型有独特的技能树

### 基地建设系统

#### 基地功能
- **资源加工**：将原始资源转化为高级材料
- **资源存储**：扩大资源存储容量
- **科研中心**：解锁新科技
- **防御系统**：保护基地免受威胁
- **贸易中心**：方便与其他玩家交易

#### 基地升级
- **建筑升级**：提升建筑效率和容量
- **基地布局**：自定义基地布局
- **特殊建筑**：解锁独特功能的特殊建筑

### 科技系统

#### 科技树
- **探索科技**：提升探索能力
- **采矿科技**：提升资源采集效率
- **飞船科技**：提升飞船性能
- **建筑科技**：提升基地功能
- **战斗科技**：提升防御和攻击能力
